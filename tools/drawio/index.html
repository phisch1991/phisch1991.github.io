Graphics in your documentation

Draw.io to the rescue

The problem

A long time during my career we did not find a good solution for managing diagrams/figures within Markdown or AsciiDoc documentation. We had a source file (e. g. a Microsoft Visio file) and then rendered a pixel file which was included into the documentation. The challenges we had include:

  • The application required to modify the source file was not installed (due to cost saving, licensing or carelessness)
  • The source file was on somebody elses computer and he/she did not upload it, so we either had to wait or (in the worst case) create the file again.
  • Even if there was a source file, there was no guarantee that the source file and the pixel file contain the same diagram (e. g. if somebody forgot to render)

The solution

We stumbled across a tool called Draw.io which is available as a web and native app (at least for MacOS). It not only allows you to draw diagrams without the need of an commercial tool, but it also allows you to only manage on file, which contains both source and pixels. This is done by embedding the source into the metadata of a pixel file such as _.png. As a result, all the challenges above where solved quite easy.
